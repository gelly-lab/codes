# シンプルなECサイト - ユーザーストーリー一覧

**最終更新：** 2026年2月17日  
**ステータス：** 仮版（ステップ5＆6の確認後に最終化）

---

## 📌 プロジェクト概要

- **プロダクト名：** シンプルなSaaS型ECサイト
- **ターゲットユーザー：** Ripperdoc（vibe codingを体感したい技術者）
- **商品数：** 約10件程度
- **対象地域：** 国内
- **決済方式：** 疑似的な決済機能

---

## 🎯 MVP（第1優先）

### US-001：商品一覧画面でサービス内容を把握したい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、トップページにアクセスしたときに、提供されているSaaSサービス一覧が見やすく表示されたい。なぜなら、どんなサービスが利用可能か素早く把握したいから。

**受け入れ基準：**
- [ ] トップページに商品（SaaS）が一覧表示される
- [ ] 各商品には、サムネイル画像、名前、簡単な説明が表示される
- [ ] 商品の並び順は管理画面で変更可能（将来機能）
- [ ] レスポンシブデザインで、スマートフォン・タブレット・PCで正常に表示される
- [ ] ページ読み込み時間が3秒以内

---

### US-002：商品詳細から詳しい情報を確認したい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、商品一覧から気になるサービスをクリックすると、詳細ページに遷移して、料金、機能説明、動作確認画面のリンクなどを確認したい。なぜなら、購入前に十分な情報を確認して判断したいから。

**受け入れ基準：**
- [ ] 商品一覧から商品をクリックすると、詳細ページに遷移する
- [ ] 詳細ページには、以下の情報が表示される：
  - 商品画像
  - 商品名
  - 説明文
  - 料金
  - 機能一覧
  - 「カートに追加」ボタン
- [ ] 「戻る」ボタンで一覧ページに戻れる
- [ ] 商品別のユニークなデモ画面へのリンクが表示される

---

### US-003：気になったサービスをカートに入れたい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、商品の一覧から、検討したいサービスを選択し
それぞれの製品の機能比較を行いたい。
データは実際のSaasサービスサイトから取り込んでほしい

**受け入れ基準：**
- [ ] 商品詳細ページの「カートに追加」ボタンをクリックするとカートに追加される
- [ ] カートに追加された際に、トースト通知で「カートに追加しました」と表示される
- [ ] ローカルストレージ（またはセッション）にカート情報が保存される
- [ ] 画面右上のカートアイコンにカート内の商品数が表示される
- [ ] 同じ商品を2回カートに追加した場合、カート内の個数が増加する

---

### US-004：カート内容を確認して注文したい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、カートアイコンをクリックしてカート画面を開き、選択した商品の比較表と
ポイント制の評価グラフ下表を閲覧したい

**受け入れ基準：**
- [ ] カート画面で、カート内のすべての商品が表示される
- [ ] 各商品について、以下が表示される：
  - 商品名
  - 単価 （適当でいい）
    - 削除ボタン
- [ ] 合計金額がカート下部に表示される
- [ ] 「チェックアウト」ボタンが表示される
- [ ] 「ショッピング続行」ボタンで一覧ページに戻れる
- [ ] 個数の変更が可能（増減可）

---

### US-005：疑似決済を行いたい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、「チェックアウト」ボタンをクリックすると、注文確認画面に遷移し、簡単な情報入力（メールアドレス）後に「注文確定」ボタンで疑似的に決済完了したい。なぜなら、ECサイトの一連の流れを体験したいから。

**受け入れ基準：**
- [ ] チェックアウト画面で以下の情報入力欄が表示される：
  - メールアドレス
  - 氏名
  - 支払い方法（リスト選択）
- [ ] 「注文確定」ボタンをクリックすると、注文完了画面に遷移する
- [ ] 注文完了画面では以下が表示される：
  - 注文番号
  - 注文日時
  - 購入商品一覧
  - 合計金額
  - 「契約書をダウンロード」ボタン（SaaS向け）
  - 「メールで送付」ボタン
- [ ] 決済は実際には行われず、ダミー決済として処理される

---

### US-006：購入後に契約書を確認したい
**ユーザーペルソナ：** Ripperdoc

**ストーリー：**
Ripperdocとして、注文完了後に「契約書をダウンロード」ボタンをクリックすると、購入サービスの契約書（またはサービス利用規約）PDFをダウンロードできたい。なぜなら、SaaS利用時に法的な確認が必要だから。

**受け入れ基準：**
- [ ] 注文完了画面に「契約書をダウンロード」ボタンが表示される
- [ ] ボタンをクリックするとPDFファイルがダウンロード開始される
- [ ] PDFには、以下の情報が含まれる：
  - 契約日
  - 顧客名
  - 購入サービス名
  - 料金
  - サービス利用規約のダミーテキスト

---

## ⭐ ユニークな機能（案）

### US-007：デモ機能でサービスをお試し利用したい
**ユーザーペルソナ：** Ripperdoc  
**優先度：** 第2優先（MVP完了後検討）

**ストーリー：**
Ripperdocとして、商品詳細ページから各SaaS製品のデモ版にアクセスしたい。なぜなら、実際の操作感を確認してから購入を決定したいから。

**受け入れ実装案：**
- 各商品のデモ画面ページへのリンク
- ダミーのインタラクティブ要素（例：簡易チャット、キャンバス、フォームなど）
- 「このサービスが気に入った場合は、上記をクリックして購入へ進む」というCTA

## 📋 非機能要件

### セキュリティ
- [ ] HTTPS対応
- [ ] XSS対策（ユーザー入力のサニタイズ）
- [ ] CSRF対策（トークン生成）

### パフォーマンス
- [ ] ページロード時間：3秒以内
- [ ] 初期化時間：1秒以内
- [ ] API応答時間：500ms以内

### ユーザーインターフェース
- [ ] レスポンシブデザイン対応
- [ ] アクセイビリティ対応（基本的なもの）
- [ ] Vibe Coding 実感できるアニメーション・インタラクション
